# 法律文档知识库功能开发计划（执行版）

## 1. 目标

按功能闭环推进开发，不按时间分期。每完成一个功能包即可联调与验收，确认后进入下一个功能包。

技术基线：
- Ubuntu 24.04
- Node.js v25.8.0 + NestJS（单体）
- PostgreSQL 16.13
- Nginx + PM2/Systemd

## 2. 功能包顺序（按依赖）

1. `FP-01` 项目与基础设施骨架
2. `FP-02` 认证与权限（管理员/游客）
3. `FP-03` 文档上传、存储、格式转换（Word/TXT/PDF/HTML -> MD）
4. `FP-04` 文档管理（元数据、归档、作废、回收站）
5. `FP-05` 版本管理（快照、对比、回滚）
6. `FP-06` 检索与预览（全文、筛选、排序、游客可见）
7. `FP-07` 审计日志与安全控制
8. `FP-08` 稳定性与上线能力（备份、恢复、启动脚本）

## 3. 功能包定义

## FP-01 项目与基础设施骨架
- 后端：NestJS 模块骨架（auth/document/search/audit/system）
- 前端：Vue3 管理端+游客端路由骨架
- 数据库：迁移机制、基础表初始化
- 完成定义：
  - 前后端可启动
  - PostgreSQL 连通
  - 健康检查接口可访问

## FP-02 认证与权限
- 管理员登录、会话/JWT、验证码
- 游客免登录访问公共接口
- 后端接口级 RBAC（管理员/游客）
- 完成定义：
  - 游客访问管理接口返回 403
  - 管理员登录态可访问管理接口

## FP-03 上传与格式转换
- 单个/批量上传
- 文件类型校验（doc/docx/txt/pdf/html）
- 异步转换为 Markdown（失败可重试）
- 原始文件与 Markdown 存储
- 完成定义：
  - 上传后可得到 Markdown 内容
  - 失败任务可查询原因并重试

## FP-04 文档管理
- 元数据 CRUD（类型、层级、领域、关键词、生效/失效日期、发布机关、文号等）
- 自动归档 + 手动调整（目录最多 5 级）
- 删除、恢复、永久删除、作废
- 完成定义：
  - 文档状态流转正确（effective/expired/void/deleted）
  - 回收站策略可执行

## FP-05 版本管理
- 每次正文/关键元数据变更自动生成版本
- 版本列表、版本差异查看
- 回滚（回滚生成新版本）
- 完成定义：
  - 任意文档可查看版本历史
  - 回滚后历史不丢失

## FP-06 检索与预览
- PostgreSQL 全文检索（FTS + pg_trgm）
- 条件筛选（层级/领域/时间/状态/发布机关/文号）
- 排序（相关性/发布时间/更新时间）
- Markdown 预览页
- 完成定义：
  - 游客可检索和预览所有公开文档
  - 游客无下载/分享入口

## FP-07 审计与安全
- 管理员关键操作审计日志
- 上传病毒扫描（ClamAV）
- 敏感操作二次确认（删除/回滚）
- 完成定义：
  - 关键操作全链路可追溯
  - 恶意文件可拦截

## FP-08 上线能力
- 自动每日备份 + 手动备份
- 恢复脚本
- Nginx 反向代理与 PM2/Systemd 启动配置
- 完成定义：
  - 服务可重启自动拉起
  - 备份恢复演练通过

## 4. 每个功能包的交付格式（固定）

每完成一个功能包，固定交付以下内容：
- 代码变更清单
- 数据库变更脚本
- API 文档更新
- 最小验证步骤（3-5 条）
- 已知限制与下个功能包依赖

## 5. 开发启动顺序（你确认后立即执行）

第一步从 `FP-01` 开始，完成后我会把可运行结果和验证步骤给你确认，再进入 `FP-02`，按此循环直到 `FP-08`。
