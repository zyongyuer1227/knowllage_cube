# 法律文档知识库开发计划（V1）

## 1. 计划目标

在已确认技术基线下完成首个可上线版本（P0），支持：
- 管理员上传/编辑/归档/版本/删除作废
- 游客检索与预览全部法律库文档
- Word/TXT/PDF/HTML 转 Markdown
- 基础审计日志与安全控制

技术基线：
- Ubuntu 24.04
- Node.js v25.8.0 + NestJS（单体）
- PostgreSQL 16.13（含全文检索）
- Nginx + PM2/Systemd

## 2. 范围定义

### 2.1 本期范围（MVP / P0）
- 角色与权限（管理员、游客）
- 文档上传与异步转换（含失败重试）
- 元数据管理与归档
- 版本管理（列表、对比、回滚）
- 删除/作废/回收站
- 检索（全文+筛选+排序）与 Markdown 预览
- 审计日志（管理员关键操作）

### 2.2 暂不进入本期
- 标签自动识别、关联推荐
- 外部 OA/法务系统集成
- 高级统计看板与报表导出

## 3. 里程碑与工期（建议 6 周）

## M0：项目初始化（第 1 周）
- 输出项：
  - 仓库结构、分支策略、CI 基础流程
  - 数据库初始化脚本与迁移机制
  - 环境配置模板（dev/test/prod）
- 验收：
  - 后端/前端可本地启动
  - PostgreSQL 连通与基础表创建成功

## M1：核心后端能力（第 2-3 周）
- 输出项：
  - 认证鉴权（管理员登录、游客访客态）
  - 文档主流程 API：上传、编辑、删除、作废、回收站、版本
  - 文档转换任务链路（文件入库 -> 转 Markdown -> 入库）
  - 审计日志记录
- 验收：
  - P0 接口联调通过
  - 关键操作均有日志

## M2：检索与前端闭环（第 4-5 周）
- 输出项：
  - Postgres 全文检索与组合筛选
  - 游客检索页、结果页、文档预览页
  - 管理员工作台（上传、编辑、版本、回收站）
  - 权限可见性与接口防越权
- 验收：
  - 游客可完整检索和预览
  - 管理员全流程可用
  - 越权测试通过

## M3：稳定性与上线（第 6 周）
- 输出项：
  - 性能压测与慢 SQL 优化
  - 备份/恢复脚本
  - Ubuntu 24.04 生产部署（Nginx + PM2/Systemd）
  - 上线手册与回滚预案
- 验收：
  - 20 并发（峰值 30）满足响应目标
  - 备份恢复演练通过
  - 可灰度发布与快速回滚

## 4. 模块任务拆解（WBS）

### 4.1 后端（NestJS）
- 用户与鉴权：管理员登录、JWT/Session、验证码校验
- 文档模块：上传、元数据、状态机（生效/失效/作废/删除）
- 版本模块：快照存储、差异对比、回滚
- 检索模块：tsvector 索引、组合查询、排序
- 审计模块：操作事件入库、查询 API
- 定时任务：回收站过期清理、每日备份触发

### 4.2 前端（Vue3）
- 游客端：检索、筛选、预览
- 管理端：登录、工作台、文档 CRUD、版本管理、回收站
- 统一能力：路由守卫、权限控制、错误提示、加载状态

### 4.3 数据与运维
- PostgreSQL 表结构、索引、迁移脚本
- 文件目录规范与存储策略
- Nginx 反向代理与 HTTPS
- PM2/Systemd 进程托管与日志轮转

## 5. 团队分工建议

- 后端开发：1 人（NestJS + PostgreSQL）
- 前端开发：1 人（Vue3）
- 测试/联调：1 人（可兼职）
- 运维支持：0.5 人（上线周投入）

> 若仅 2 人团队，建议“后端主导 + 前端兼测”，总工期建议 7-8 周。

## 6. 验收标准（与需求文档对齐）

- 功能验收：
  - P0 全通过，阻断级缺陷为 0
- 性能验收：
  - 简单检索 P95 <= 0.5s
  - 全文检索 P95 <= 1s
  - 20 并发稳定，峰值 30 可接受
- 安全验收：
  - 游客无法访问管理接口（403）
  - 上传文件经过病毒扫描
  - 审计日志可追溯关键操作

## 7. 风险与缓解

- 文档转换准确率不稳定：
  - 缓解：先覆盖高频格式，建立失败回退和人工修订流程
- 检索性能波动：
  - 缓解：优先建索引、分页与查询约束、慢 SQL 监控
- 上线后权限漏洞：
  - 缓解：接口级权限测试清单 + 上线前渗透检查

## 8. 首周执行清单（可立即开工）

1. 初始化仓库目录与编码规范
2. 完成 PostgreSQL 建表与迁移框架
3. 搭建 NestJS 基础模块（auth/document/search/audit）
4. 搭建 Vue3 管理端与游客端路由骨架
5. 打通“上传 -> 转 Markdown -> 预览”最小链路
