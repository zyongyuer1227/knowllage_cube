- 上传并转换 `uploadAndConvert`

- 编辑文档元数据 `updateDocument`

## 8. 2026-03-31 补充记录（保存后最终结果不可见）

本节补记本轮后续刚处理的一个具体问题，避免下次继续开发时把它误判成“导入没成功”或“属性没保存”。

### 8.1 问题现象

现象描述：

- 导入文档后，在管理员界面选择属性，可以看到当前编辑态内容正常显示
- 点击“保存源码”后，管理员侧看不到“最终结果”或看到的仍是本地临时效果
- 用户会误以为保存后的最终产物没有正确生成

### 8.2 排查结论

实际排查后确认：

- 后端文档保存与资产落盘本身是正常的
- `apps/backend/storage/documents/43/metadata.json` 中已能看到：
  - `businessPath`
  - `legalPath`
  - `previewHtmlPath`
- `content.md`、`content.txt`、`preview.html` 都已存在

因此问题不在于：

- 属性没有入库
- Markdown 没有保存
- 预览 HTML 没有生成

真正的问题在于前端展示链路不一致：

- 游客页 `GuestPage.vue` 会优先使用后端落盘的 `previewHtml`
- 管理员页 `AdminPage.vue` 之前只使用 `workspace.editorMarkdown` 做前端即时渲染

这会导致：

- 管理员页看到的是“编辑态临时渲染”
- 游客页/最终结果看到的是“后端持久化产物”
- 两边渲染源不一致时，就会出现“保存后无法看到最终结果”的错觉

### 8.3 已做修复

已修改：

- `apps/frontend/src/views/AdminPage.vue`

修复方式：

- 管理员页中的 `GovDocPreview` 现在也传入 `workspace.activeDocument.previewHtml`
- 当后端已有持久化 HTML 时，管理员预览与游客页统一走同一份最终产物
- 只有在没有持久化 HTML 时，才退回前端基于 Markdown 的即时渲染

这次修复的本质是统一“管理员预览”和“最终展示”的渲染源，避免双轨显示。
