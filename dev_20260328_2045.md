## 2. 本轮完成的核心开发

### 2.1 文档持久化增强

已将文档相关的本地持久化链路补齐，不再只依赖数据库中的 `markdown_content`：

- 上传原件保存到：
  - `storage/documents/<documentId>/source/`
- 转换后 Markdown 落盘：
  - `storage/documents/<documentId>/content.md`
- 转换后纯文本落盘：
  - `storage/documents/<documentId>/content.txt`
- 渲染后 HTML 快照落盘：
  - `storage/documents/<documentId>/preview.html`
- 文档资产元数据落盘：
  - `storage/documents/<documentId>/metadata.json`

后端在以下场景会自动同步这些资产：

- 上传并转换成功后
- 编辑正文后
- 回滚版本后
- 更新文档标题或归档路径后

### 2.2 转换任务恢复

后端已支持在服务启动时自动扫描并恢复 `pending` / `processing` 状态的转换任务，避免服务中断后任务悬空。

实现位置：

- `apps/backend/src/modules/document/document.service.ts`

### 2.3 欢迎页改造成系统持久化文档

“欢迎.md” 已不再是前端 `localStorage` 草稿，也不在游客文档树中出现。

当前设计为：

- 游客不可见、不可点击、不可选
- 游客未选中文档时，右侧使用它作为兜底欢迎页展示
- 管理员未选中真实文档时，右侧默认编辑的是这份欢迎文档
- 管理员保存后，写入服务器本地文件

实际落盘文件：

- `storage/system/welcome.md`
- `storage/system/welcome.json`
- `storage/system/welcome.preview.html`

对应接口：

- `GET /api/v1/system/welcome-document`
- `GET /api/v1/system/admin/welcome-document`
- `PUT /api/v1/system/admin/welcome-document`

### 2.4 管理员参数设置弹窗误关闭修复

已修复管理员“参数设置”弹窗在文本框中点击、拖拽选中内容时误关闭的问题。

现在行为改为：

- 只有在遮罩空白处按下并抬起，才会关闭弹窗
- 在弹窗内容区内点击、输入、选中文本，不会触发关闭

### 2.5 文档拖拽移动修复

已修复管理员界面中“拖拽文档到其他文件夹时只有动画、实际不移动”的问题。

处理方式：

- 对 `dataTransfer` 的读取增加标准类型兜底
- drop 时阻断事件冒泡
- 保证拖拽数据在不同浏览器 / 不同 DOM 命中情况下仍能读取到文档标识

### 2.6 “一键转换MD格式”改为 Python + 大模型

原先前端按钮只是本地调用 `normalizeToMarkdown()` 进行简单规整。  
现在已改为：

1. 管理员把原始文字粘贴进编辑区
2. 点击“一键转换md格式”
3. 前端请求后端接口：
   - `POST /api/v1/admin/documents/format-text-import`
4. 后端调用 Python 脚本：
   - `scripts/text2md_formatter.py`
5. Python 脚本通过大模型整理文本为 Markdown 风格 TXT
6. 后端再将生成结果接入现有文档落库 / 版本 / 预览链路

### 2.7 一键转换支持“就地格式化当前文档”

用户实际使用方式通常是：

1. 先点击“新增文档”
2. 输入标题
3. 粘贴原始文本
4. 点击“一键转换md格式”

此前新逻辑会再次尝试创建同名文档，触发标题唯一冲突。  
现在已改成：

- 如果当前选中的是已存在文档：
  - 直接将大模型格式化结果写回当前文档
- 如果当前不是已持久化文档：
  - 才走“导入为新文档”的流程

这解决了“先新增占坑文档再格式化”时的同名冲突问题。

### 2.8 文档标题唯一约束

现在后端已强制要求文档标题唯一：

- 新增文档时，不允许与现有文档同名
- 大模型格式化导入时，不允许创建同名新文档
- 修改文档标题时，不允许改成其他已存在文档的标题
- 自己保留原标题是允许的

错误形式类似：

```json
{
  "message": "Document title already exists: xxx",
  "error": "Bad Request",
  "statusCode": 400
}
```

## 7. 当前行为边界

### 欢迎文档

- 是系统级本地文件，不是数据库中的普通文档
- 游客不能在树中看到它
- 管理员可以编辑并保存它
- 它用于游客未选中文档时的兜底展示

### 文档标题

- 标题现在视为唯一键级别的业务约束
- 后续如果要支持“同名不同目录”，需要重新调整这条规则和相关查询逻辑

### 一键转换MD格式

- 依赖后端能成功调用 Python
- 依赖模型 API Key 配置正确
- 当前默认策略是：
  - 真实文档就地写回
  - 非真实文档则新建导入
